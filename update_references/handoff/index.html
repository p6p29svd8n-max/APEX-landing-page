01 Overview

Ship the V1 Quant direction, dark, compact density.

This is the handoff brief for the Aurealis landing-page rebuild. The visual reference is Landing Page.html in the project root — open it alongside this doc to see every section rendered. Only V1 (Quant Terminal, dark theme) at compact density ships. V2 (Editorial) and V3 (Research) were exploratory and should not be ported.

Design intent

Bloomberg-terminal vibe for sophisticated retail investors. Mono labels, tight data grids, scientific precision, teal (#00d4aa) as the only chromatic accent. No gradients, no glassmorphism, no rounded cards. Hairline borders, 1px everywhere.

Primary goals

  • Funnel qualified users to the #beta form (primary CTA).
  • Communicate institutional-grade tooling in under 10 seconds above the fold.
  • Showcase MVP feature set clearly separated from post-beta roadmap.
  • Feel fast, weightless, and respectful of the reader's time.

Non-goals

  • Marketing illustrations, stock photos, AI-generated imagery.
  • Live market data — all tickers are static visual chrome.
  • Dark/light toggle — the page is dark-only for V1.

Non-negotiable

Teal accent #00d4aa is the single chromatic accent and must be used sparingly — CTAs, active states, positive deltas, hairline focus rings. Never as large background fills. Never gradient.

Scale & density

Container max
1200px, 32px gutter
Section padding
72px top/bottom · compact density
Grid gap
1px (hairline dividers) or 24px (cards)
Border radius
0 — everything square. Zero rounding.
Border width
1px solid. Never 2px. Never dashed except focus.
Shadows
None. Elevation via border + surface shift only.

Motion

  • Scroll reveals: 28px translateY + opacity, 900ms cubic-bezier(0.16,1,0.3,1). Stagger children with .d1, .d2, .d3, .d4 (80ms steps).
  • Hero background: canvas-drawn drifting grid + pulsing data nodes. Low opacity, never distracting. Pause on prefers-reduced-motion.
  • Ticker ribbon: infinite horizontal scroll, 60s linear cycle, duplicate content for seamless loop.
  • Hover: 150ms color/border transitions only. No lift, no scale.

Canonical copy lives in landing/content.jsx. The split below reflects the agreed MVP scope — current features ship with beta, planned features are roadmap. Do not reword without sign-off.

    Curate your highest-conviction picks with live prices, key figures, fair-value estimates and AI recommendations in one focused view.
  • AI Equity Research Reports file-text

  • Stock Viewer bar-chart-2
    Live charts, financial & business data, short-interest, insider transactions, analyst consensus, and news in one panel.
  • Recession Radar activity
    11 macro signals merged into one composite score across 5 regimes. Understand recession risk and market sentiment at a glance.
  • Portfolio AI Analysis briefcase
    Monitor performance, sector allocation and value history. Challenge your positions via critical AI-based portfolio analysis.
  • Plug & Play DCF Analyzer calculator
    Build comprehensive DCF models with adjustable WACC and terminal growth. AI commentary on key value drivers in under 30 seconds.

Planned features POST-BETA · 3

  • Sector Insights layers
    Evaluate and filter current and historical performance across prominent sectors, with insight into leading companies in each.
  • Macro, World News & Earnings globe
    Yield curves, FX, commodities, global news, upcoming earnings and an estimated fear & greed index — in one tab.
  • Trend Intelligence search
    Vertical and horizontal trend analysis to identify market movements and investment opportunities across supply-chains and sectors.

06 Implementation notes

How to port this into the real codebase.

Stack recommendation

  • Next.js 14 (App Router) or Vite + React 18 — either works. Landing is a single static route.
  • CSS: keep tokens as CSS variables on :root (as in the reference). Tailwind optional; not required.
  • Icons: lucide-react — the icon key in content maps 1:1 to Lucide names.
  • Fonts: self-host Inter + JetBrains Mono via next/font or @fontsource. Don't rely on Google Fonts CDN in prod.

Component split (suggested)

// app/(marketing)/page.tsx
import { Nav, Hero, Ticker, Product,
         Features, How, Sample, Pricing,
         Beta, Footer } from '@/components/landing';

export default function Landing() {
  return (
    <>
      <Nav />
      <Hero />        {/* canvas bg + copy */}
      <Ticker />
      <Product />     {/* FIG.01 screenshot */}
      <Features />    {/* MVP + Planned */}
      <How />         {/* 4 steps */}
      <Sample />      {/* 9-section report */}
      <Pricing />     {/* 3 tiers, mo/yr toggle */}
      <Beta />        {/* email form */}
      <Footer />
    </>
  );
}

Token file

Lift the .v1 block from Landing Page.html (lines ~24–49) into styles/tokens.css and scope to :root. Drop the .v1 class prefix — V1 is now the only theme.

Content source

landing/content.jsx is already structured. Copy it as-is to lib/landing-content.ts, convert the icon-name strings into Lucide imports, done.

Hero canvas

The animated hero background is a small canvas component (~80 lines). See V1HeroBg in Landing Page.html. It draws a slowly-drifting dot grid with randomly-pulsing accent nodes. Port verbatim; respect prefers-reduced-motion.

Accessibility

  • All .mono labels must still meet WCAG AA against their surface (they do at 12px #8a97a6 on #0a0d12; verify if darkened).
  • Ticker must pause on focus/hover per WCAG 2.2.2.
  • Focus rings: 1px dashed #00d4aa with 2px offset. Never remove.
  • Hero canvas: aria-hidden="true".

Performance

  • Landing should score ≥95 Lighthouse Performance on mobile.
  • Hero screenshot: serve AVIF + WebP fallback, loading="eager", fetchpriority="high".
  • Feature screenshots: lazy-load, loading="lazy".
  • No client-side routing on landing — single static page.
07 Acceptance checklist

Done when all of this is true.

  • Renders identically to Landing Page.html at 1440px, 1024px, 768px, 375px.
  • All anchor links (#features, #how, #pricing, #beta) smooth-scroll with ~60px offset for the fixed nav.
  • Hero animated background runs at 60fps on a 2020 MacBook Air; pauses with prefers-reduced-motion.
  • Ticker loops seamlessly — no visible reset.
  • Pricing monthly/annual toggle updates all three tiers and the footnote.
  • Beta form submits to the real endpoint, shows inline success state, errors inline.
  • Lighthouse: Perf ≥95 · A11y ≥95 · Best Practices 100 · SEO 100.
  • No console errors, no CLS on hero.
  • OG image + meta tags set (title: "Aurealis — Institutional-grade research for retail investors").
